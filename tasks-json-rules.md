# Структура JSON и правила заполнения

## Назначение
Файл `data/tasks.json` хранит справочник заданий для теста по орфографии.  
Корневой тип: массив объектов (`Task[]`).

## Формат одного задания (`Task`)
```json
{
  "id": "t01",
  "wordMask": "б..рег",
  "correctWord": "берег",
  "correctLetter": "е",
  "options": ["а", "о", "е"],
  "ruleType": "unstressed_vowel",
  "hint": "Проверка: берег — берега."
}
```

## Обязательные поля
- `id` (`string`): уникальный идентификатор задания.
- `wordMask` (`string`): слово с ровно одним пропуском `..`.
- `correctWord` (`string`): правильное написание слова полностью.
- `correctLetter` (`string`): одна буква, вставляемая вместо пропуска.
- `options` (`string[]`): массив из 2-4 букв (каждый элемент длиной 1).
- `ruleType` (`string`): тип орфограммы.
- `hint` (`string`): краткое пояснение при ошибке.

## Допустимые значения `ruleType`
- `unstressed_vowel`
- `paired_consonant`
- `zhi_shi`
- `cha_shcha`
- `chu_shchu`
- `dictionary`

## Правила заполнения
1. В `wordMask` должен быть ровно один фрагмент `..`.
2. `options` обязательно содержит `correctLetter`.
3. В `options` только буквы, без слогов и слов.
4. `correctLetter` и каждый элемент `options` имеют длину 1.
5. `id` не повторяется в пределах файла.
6. `hint` соответствует `ruleType`:
   - для проверяемых орфограмм: краткая проверка/правило,
   - для словарных слов: формат `Словарное слово. Нужно запомнить.`
7. Используются слова уровня 2 класса.

## Рекомендации по качеству данных
- Держать единый стиль формулировок (`Проверка: ...`).
- Избегать редких и сложных слов.
- Проверять файл после правок:
  - валидность JSON,
  - отсутствие дубликатов `id`,
  - корректность `wordMask`/`correctLetter`/`options`.
