# Структура JSON и правила заполнения

## Назначение
Словари лежат в `data/dictionaries/*.json`.  
Каждое задание поддерживает одну или несколько орфограмм внутри одного слова.

## Формат словаря
```json
{
  "title": "2 класс — основной словарь",
  "description": "Полный набор орфографических заданий",
  "tasks": [
    {
      "word": "м[ао|о]л[ао|о]ко",
      "hint": "Проверка: молоко — молочный."
    }
  ]
}
```

## Формат задания
- `word` (`string`) — слово-шаблон с 1+ орфограммами.
- `hint` (`string`) — одно общее пояснение для всей задачи.

`id` не используется.

## Синтаксис `word`
- Орфограмма задается токеном: `[варианты|правильная]`
- Пример: `м[ао|о]л[ао|о]ко`
- Где:
  - `варианты` — строка из 2-4 букв
  - `правильная` — ровно одна буква

## Правила валидации
1. В `word` должен быть минимум один токен `[...]`.
2. Каждый токен строго в формате `[варианты|правильная]`.
3. `правильная` должна входить в `варианты`.
4. Количество вариантов в токене: от 2 до 4.
5. `hint` обязателен и не пустой.

## Поведение в приложении
1. Для каждой орфограммы показываются свои варианты букв.
2. Пользователь выбирает буквы последовательно: первая орфограмма, вторая и т.д.
3. Проверка выполняется один раз после выбора всех букв.
4. Показывается общий итог (`Верно/Ошибка`) и один общий `hint`.
